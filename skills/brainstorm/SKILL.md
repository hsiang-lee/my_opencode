# Brainstorm — 设计探索方法

通过对话澄清需求、探索方案、逐步确认设计。

## 提问澄清

聚焦理解四件事：

| 维度 | 问题 |
|------|------|
| 目的 | 为什么做这个？解决什么问题？ |
| 约束 | 时间、技术栈、兼容性要求？ |
| 成功标准 | 怎么知道做对了？ |
| 范围 | 什么明确不做？ |

**方式：** 优先选择题，也欢迎开放式回答。

**反模式：** 一次问多个、模糊问题、跳过问题直接设计。

---

## 提出方案

提出 **2-3 个不同方案**，每方案含：
- 描述
- 优势
- 劣势

**YAGNI：** 删除不必要功能，MVP 优先。

---

## 展示设计

逐部分展示，每部分后确认。覆盖：

| 要素 | 内容 |
|------|------|
| 架构 | 组件划分、接口定义 |
| 数据流 | 从哪来到哪去 |
| 组件 | 职责、接口、依赖 |
| 错误处理 | 异常路径、边界条件 |

## 红旗 — 停止

- 一次问多个问题
- 用户没批准就推进
- 设计包含"TBD"或"稍后决定"
- 没有备选就说"我知道最佳方案"
