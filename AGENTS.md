## Agent 使用指南 (新增)

| Agent | 功能 | 使用时机 | 调用方式 |
|-------|------|----------|----------|
| **CREATIVE** | 新功能开发强制流程 | 新功能开发 (设计→计划→实现→审查→收尾) | Tab 切换到 CREATIVE |
| **Debug** | Bug修复强制流程 | Bug 修复 (复现→根因→TDD修复→审查→收尾) | Tab 切换到 Debug |

## CREATIVE 工作流 Agent

**使用场景:** 新功能开发，需要结构化流程保证质量

**流程:** 设计探索 → 设计文档 → 实现计划 → 计划评审 → TDD执行 → 代码审查 → 分支收尾 (7步，不可跳过)

**Step 1 设计探索:** self(brainstorm) + 可选并行 explore/librarian

## Debug 工作流 Agent

**使用场景:** Bug 修复，需要系统化根因调查

**流程:** 复现确认 → 根因调查 → 假设验证 → TDD修复 → 审查 → 收尾 (5步，不可跳过)

**与 build/plan 的区别:**
- build: 灵活直接，适合小改动和探索
- plan: 只读分析，适合阅读和规划
- CREATIVE: 新功能强制流程
- Debug: Bug修复强制流程
